01 — GROUND TRUTH

Quantifies placement repeatability so checkpoint error is not reported as more precise than the physical reference permits.

1 SESSION · 10 PLACEMENTS
Scenario / ID Purpose Setup & Controls Step-by-step Procedure Evidence, Metrics & Pass Criteria
Ground-truth repeatability GROUND_TRUTH_REPEAT
Condition: nominal 		Measure the uncertainty introduced when the robot is repeatedly placed on one surveyed floor marker.
  • Eight surveyed markers remain fixed across both rooms.
  • Select one marker for all ten placements.
  • Do not change its X, Y, or yaw reference.
  • Keep map, firmware, and localizer parameters frozen.
  1. Run preflight and create a ground_truth session.
  2. Start capture and mission; wait for a valid stable pose.
  3. Place the robot on the selected marker and record the placement.
  4. Remove the robot completely from the marker.
  5. Reposition it independently and record again.
  6. Repeat until exactly ten placements are stored.
  7. Stop, analyze, review, and finalize.
  • Exactly 10 valid placement records.
  • Position spread.
  • Heading spread.
  • Declared ground-truth uncertainty.
  • Pass: one unchanged marker reference is used throughout.

03 — KIDNAPPED

Measures global relocalization after the robot is manually moved to an unknown pose. The destination reference is recorded by the evaluator only and is never sent to the localizer.

20 TRIALS · 60-S TIMEOUT
Scenario / ID Purpose Setup & Controls Step-by-step Procedure Evidence, Metrics & Pass Criteria
Same-room relocationKIDNAP_SAME_ROOM
10 trials 		Evaluate recovery after a moderate relocation within the current room.
  • Preselect start marker A and destination marker B in one room.
  • Freeze destination position and orientation for the trial set.
  1. Create a kidnapped session and start capture/mission.
  2. At A, wait until TRACKING is stable.
  3. Press KIDNAP START.
  4. Safely cover the LiDAR view; do not power the sensor off.
  5. Manually move the robot to marker B and set the planned yaw.
  6. Select B as evaluator ground truth only.
  7. Uncover LiDAR and press KIDNAP RELEASE + REFERENCE.
  8. Wait up to 60 seconds for global search and stable tracking.
  9. Record the final checkpoint, then stop and analyze.
  • Relocalization success within 60 seconds.
  • First scan after release.
  • Median and P95 recovery time.
  • Final position and heading error.
  • Failure count and final score.
  • Pass: one release event and one final checkpoint exist.
Cross-room relocationKIDNAP_CROSS_ROOM
10 trials 		Evaluate the harder case where the robot is moved into the other room.
  • Place A and B in different rooms.
  • Use the same destination marker/yaw plan across repetitions.
  • Return the LiDAR view to normal before release.

Safety: STOP MISSION is not a mechanical emergency stop. Secure the AGV against unintended motion before manually lifting or relocating it.

04 — DYNAMIC OCCLUDED

Produces one simple, repeatable human-motion disturbance. One person crosses the robot path at the same trigger point in every trial.

10 TRIALS · 3 ± 0.5 S CROSSING
Scenario / ID Purpose Setup & Controls Step-by-step Procedure Evidence, Metrics & Pass Criteria
Person crossingdynamic_occluded
5× R1 + 5× R2 		Measure checkpoint localization robustness during a short, controlled moving occlusion.
  • Mark pedestrian endpoints H1 and H2.
  • Mark robot trigger point T0.
  • H1–H2 crosses the robot route perpendicularly.
  • Maintain at least 1 m separation.
  • Use the same person and robot speed when possible.
  • No other people may enter the test area.
  1. Select dynamic_occluded and R1 or R2.
  2. Create the session, start capture/mission, and press ROUTE START.
  3. For R1, the person waits at H1; for R2, wait at H2.
  4. When the robot reaches T0, press PERSON START as crossing begins.
  5. Walk normally without stopping.
  6. At the opposite endpoint, press PERSON END.
  7. Confirm the displayed duration is 2.5–3.5 seconds.
  8. Record all eight checkpoints and finish the route normally.
  9. Repeat five times in each direction.
  • dynamic_occlusion.csv.
  • Crossing direction and duration.
  • Scan count and accepted-scan rate during crossing.
  • Localization score during crossing.
  • Standard route accuracy metrics.
  • Pass: exactly one START/END pair and 8 checkpoints.
  • Repeat with a new session if duration or safety fails.
